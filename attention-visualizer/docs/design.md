# Attention Visualizer Redesign

Personal learning tool for building intuition about transformer attention mechanisms — both how weights are computed (Q/K/V math) and what the resulting patterns mean.

## Architecture

Single self-contained HTML file. Three coordinated panels in a top/bottom split layout (~35% top, ~65% bottom):

1. **Text Display (top, full width)** — interactive token sequence with SVG attention lines on hover. Head selector buttons above the text.
2. **Heatmap Matrix (bottom left, ~50% width)** — N×N attention grid. Rows = query tokens, columns = key tokens. Upper triangle greyed out (causal mask) with tooltip explanation. Clickable cells.
3. **Computation Panel (bottom right, ~50% width)** — shows the Q/K/V → dot product → scale → softmax pipeline for a selected token pair.

All three panels update in response to user interaction on any panel.

**Demo sentence:** `"The cat sat on the warm mat and purred softly"` (10 tokens — keeps heatmap compact at 10×10).

## Data Layer

### Synthetic Q/K/V System

Each token gets an 8-dimensional embedding vector generated by a seeded PRNG (e.g., mulberry32 with a hash of the token string as seed), producing floats in [-1, 1], then normalized to unit length. Deterministic: same token always produces the same embedding.

Three weight matrices per head (8×8 each) project embeddings into Q, K, V spaces. V is included in the interface for completeness but not used in the current visualization (it would be needed to show weighted output vectors). Two heads with distinct matrices:

- **Head 1 (positional):** weight matrices tuned to produce attention concentrated on nearby tokens — demonstrates local/syntactic attention patterns (diagonal band in heatmap).
- **Head 2 (semantic):** weight matrices hand-tuned for the specific demo sentence so that semantically related tokens (e.g., "cat"→"purred", "mat"→"warm") produce high dot products. This head's matrices are crafted for the fixed demo sentence, not generalizable. Demonstrates content-based attention (scattered clusters in heatmap).

### Computation Flow

For query token i attending to key token j:

1. Q_i = embedding_i × W_q (8-dim vector)
2. K_j = embedding_j × W_k (8-dim vector)
3. raw_score = Q_i · K_j (scalar, dot product)
4. scaled_score = raw_score / √8
5. Apply causal mask (set scores for j > i to -∞)
6. attention_weights = softmax(scaled_scores) over all valid j

Weights are always derived from this pipeline, never hardcoded.

### Pluggable Interface

The computation module consumes:

```
{
  tokens: string[],
  embeddings: number[][],       // (n_tokens × d_model)
  heads: [{ Wq, Wk, Wv }]      // each W is (d_model × d_model)
}
```

Swapping in real model outputs means replacing this object.

## Interaction Model

### Hover on text token

- Highlights hovered token as query (bright accent)
- Draws attention lines from query to all attended tokens (line thickness ∝ weight)
- Highlights corresponding row in heatmap
- Computation panel shows summary: full softmax output vector for that query

### Click heatmap cell

- Highlights query token (row) and key token (column) in text display
- Draws single attention line between the pair
- Computation panel drills into that specific pair:
  - Q and K vectors displayed as horizontal labeled bars (8 values each, color-coded)
  - Dot product shown as single highlighted scalar
  - Scaling step: raw_score / √d_k
  - Softmax distribution as a small horizontal bar chart with the selected pair's weight highlighted

### Head selector

- Pill buttons above text display: "Head 1 (positional)" / "Head 2 (semantic)"
- Switching heads recomputes all weights and updates all three panels
- Heatmap pattern visibly changes (diagonal band vs scattered clusters)

### Causal mask

- Upper triangle of heatmap greyed out
- "Causal Mask" label with "?" hover tooltip: "Each token can only attend to itself and earlier tokens (prevents seeing the future)"

### Hover vs click state

Two interaction modes with distinct persistence:

- **Hover state (ephemeral):** hovering a text token or heatmap row shows attention for that query. Clears when the mouse moves away.
- **Click state (persistent):** clicking a heatmap cell locks the drill-down view. Persists until the user clicks another cell, clicks the same cell again to deselect, or presses Escape.

Click state takes precedence over hover state. When a cell is clicked, hovering other tokens does not change the view.

### Reset

- Hover highlights clear on mouse-leave
- Click state clears on re-click, different click, or Escape

## Visual Design

Dark mode technical aesthetic:

- **Background:** dark (#1a1a2e)
- **Text:** light (#ccc default, white for emphasis)
- **Active token:** bright accent (#e94560)
- **Attended tokens:** orange border (#f5a623), thickness ∝ weight
- **Heatmap scale:** sequential black → orange → yellow (low → high attention)
- **Causal mask cells:** dark grey (#333) at low opacity
- **Computation panel:** monospace font, color-coded vectors (cyan for Q, green for K), orange for computed values
- **Attention lines:** orange/red gradient, opacity and thickness ∝ weight
- **Head selector pills:** active = accent color, inactive = dark grey

## Corrections from Original

1. **Weights normalized via softmax** — all attention weights derived from Q/K/V computation, always sum to 1.0
2. **Token 0 has self-attention** — every token attends to at least itself
3. **Consistent self-attention** — all tokens include self in their attention distribution
4. **SVG tracks container height** — prevents line clipping on taller layouts

## Out of Scope

- Editable vector sliders
- Real model weight loading (interface is ready, data is not)
- User-typed sentences
- Multi-layer visualization
- Mobile responsiveness
